Module de paiement sécurisé Stripe pour Zen Cart v3.0.0

Introduction

Ce module de paiement Zen Cart est utilisé pour les transactions de paiement sécurisé Stripe. Vous devez signer un contrat avec la société Stripe pour recevoir les clés publiables et secrètes nécessaires au fonctionnement du module. Stripe prend en charge le traitement des paiements dans plus de 135 devises.

Supporte actuellement Zen Cart version 1.5.8 et ultérieure et PHP 8.0.0 et ultérieure. Également validé avec One-Page Checkout v2.5.5 et le template ZCA Bootstrap v3.7.8.

Notes :

  1. Nous n’assurons pas les dommages monétaires causés par ce module, vous devez donc vérifier le module en utilisant le mode de fonctionnement Test et confirmer qu’il fonctionne normalement même après le test !
  2. Le système de Stripe multiplie par 100 les devises avec 2 décimales, comme USD et EUR, avant de les envoyer aux serveurs de Stripe. Le serveur Stripe convertit le numéro reçu en 1/100 et le gère. Le yen japonais n’a pas de point décimal, donc le nombre est envoyé aux serveurs Stripe tel quel. Ce module convertit automatiquement pour chaque devise, mais le teste avec toutes les devises que vous utilisez avant de l’utiliser.
  3. Si vous ouvrez le tableau de bord Stripe => Paiements => Toute la page, vous trouverez de nombreux paiements incomplets. Parce que le serveur Stripe crée des pi_xxxxxxxxxxxxx chaque fois que la page de confirmation d’extraction du panier Zen s’affiche. et le serveur Stripe crée deux pi_xxxxxxxxxxxxx pour chaque paiement. La cause est due à la spécification de stripe server et Zencart.
  4. Si votre site utilise Paiement sur une page (OPC), vous doit add rayure à la liste séparée par des virgules des méthodes de paiement nécessitant une confirmation (Configuration : : Paramètres de paiement sur une page : : Méthodes de paiement nécessitant une confirmation).

Installation et mise à niveau

Mise à niveau d’une version antérieure vers la v3.0.0

v3.0.0 et plus tard du module de paiement est très différent des versions antérieures à v3.0.0. Il n’y a plus d’écrasement de fichier core ou de modifications de fichier modèle nécessaires (ou souhaitées).

Lors de la mise à niveau vers v3.0.0 et versions ultérieures depuis une version antérieure à v3.0.0, vous devez d’abord supprimer divers fichiers et modifications requis par ces versions précédentes. Voir cet article du wiki GitHub pour plus de détails.

IMPORTANT : Après avoir modifié la clé API ou l’état du mode de test du module de paiement, vous devez effacer tous les enregistrements existants dans le tableau de la base de données du module de paiement :

  1. Accédez aux outils de l’administrateur : : Installer l’outil SQL Patches.
  2. Entrez ce qui suit dans la zone de texte, puis appuyez sur le bouton « Envoyer » :

    TRUNCATE stripe ;

Installer le module de paiement

Pour effectuer une installation initiale du module de paiement :

  1. Décompressez le fichier zip de distribution du module.
  2. Renommer le YOUR_TEMPLATE répertoire pour correspondre au nom de votre modèle actuellement actif : /includes/template/YOUR_TEMPLATE.
  3. Copiez tous les fichiers et sous-répertoires de la distribution /comprend répertoire vers votre site /comprend répertoire.
    1. /includes/classes/ajax/zcAjaxStripe.php
    2. /includes/classes/observers/auto.stripe.php
    3. /includes/extra_datafiles/stripe_database_names.php
    4. /includes/languages/english/extra_definitions/lang.stripe_extra_definitions.php
    5. /includes/languages/english/modules/payment/lang.stripe.php
    6. /includes/languages/french/extra_definitions/lang.stripe_extra_definitions.php
    7. /includes/languages/french/modules/payment/lang.stripe.php
    8. /includes/languages/german/extra_definitions/lang.stripe_extra_definitions.php
    9. /includes/languages/german/modules/payment/lang.stripe.php
    10. /includes/languages/japanese/extra_definitions/lang.stripe_extra_definitions.php
    11. /includes/languages/japanese/modules/payment/lang.stripe.php
    12. /includes/modules/pages/account/jscript_stripe_account_form.php
    13. /includes/modules/pages/checkout_confirmation/jscript_stripe_checkout_confirmation.php
    14. /includes/modules/pages/checkout_one/jscript_stripe_checkout_one.php
    15. /includes/modules/pages/checkout_one_confirmation/jscript_stripe_checkout_one_confirmation.php

    20. /includes/templates/YOUR_TEMPLATE/css/stylesheet_stripe.css
    21. /includes/checkout.js
  4. Accédez aux modules de l’administrateur : : Paiement. Vous verrez le Stripe Payments : Carte de crédit module ajouté. Sélectionnez ce module et Installer it.
    • Remarque : Le mode de test est la valeur par défaut lorsque ce module est installé. Après le test, n’oubliez pas de mettre le mode test sur "false". Ne divulguez pas la clé secrète fournie par Stripe.

Mise à niveau du module de paiement

Suivez les étapes A à C dans la section précédente (installation).

Test sur la vitrine (Mode Test)

Utilisez ces étapes pour vous assurer que vous avez correctement installé et configuré le module de paiement :

  1. Placez un ou plusieurs articles dans votre panier et commencez le processus de paiement.
  2. Choisissez le module Stripe Payment : Credit Cardpayment et passez à l’étape de confirmation du paiement.
  3. Le formulaire de saisie de carte Stripe s’affichera comme ci-dessous. Si le formulaire n’apparaît pas, il se peut que vous ayez saisi une clé API incorrecte.

  4. Entrez le numéro de carte de crédit VISA test 4242 4242 4242 4242, une date d’expiration qui est dans le futur, un nombre aléatoire de 3 chiffres dans le CVC et le code postal (si demandé). Appuyez sur le bouton « Confirmer la commande ».
  5. Allez dans votre tableau de bord Stripe et cliquez sur le bouton Payments and Succeeded (Paiements et réussis). Vérifiez le montant total des informations, le nom du client et l’adresse e-mail.
  6. Confirmez l’adresse e-mail et le total de la commande sur les clients de l’administrateur du panier Zen : : Ordre. Comparez le montant total du panier zen, le nom du client et l’adresse e-mail avec les informations de Stripe.

Supprimer le module de paiement

Accédez à l’admin Modules : : Paiement, sélectionnez le Stripe Payments : Carte de crédit module et cliquez dessus Désinstaller bouton.

Supprimer les enregistrements Stripe des clients

Les informations de carte de crédit du client sont cryptées et enregistrées dans la base de données, mais peuvent être supprimées à la discrétion du client. Le client peut le supprimer avec le bouton de suppression des informations de carte de crédit Stripe sur la page « Mon compte ». Après la suppression, cette indication ne sera plus affichée.
