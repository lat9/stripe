Stripe Secure Payment Module v3.0.1 für Zen Cart

Einführung

Dieses Zen Cart-Zahlungsmodul wird für sichere Stripe-Zahlungstransaktionen verwendet. Sie müssen sich bei der Firma Stripe anmelden und einen Vertrag abschließen, um die veröffentlichbaren und geheimen Schlüssel zu erhalten, die für den Betrieb des Moduls erforderlich sind. Stripe unterstützt die Verarbeitung von Zahlungen in 135+ Währungen.

Derzeit unterstützt Zen Cart Version 1.5.8 und höher und PHP 8.0.0 und höher. Auch validiert mit One-Page Checkout v2.5.5 und der ZCA Bootstrap Vorlage v3.7.8.

Anmerkungen:

  1. Wir versichern nicht für irgendwelche monetären Schäden, die durch dieses Modul verursacht werden. Daher müssen Sie das Modul mit der Test-Modus-Operation überprüfen und bestätigen, dass es auch nach dem Testen normal funktioniert!
  2. Das Stripe's-System multipliziert Währungen mit 2 Nachkommastellen, wie USD und EUR, um 100, bevor es sie an die Stripe's-Server sendet. Der Stripe-Server wandelt die empfangene Zahl in 1/100 um und verwaltet sie. Der japanische Yen hat keine Dezimalstelle, daher wird die Zahl an die Stripe-Server gesendet. Dieses Modul konvertiert automatisch für jede Währung, aber testen Sie es mit allen Währungen, die Sie verwenden, bevor Sie es verwenden.
  3. Wenn Sie das Stripe-Dashboard => Zahlungen => Alle Seite öffnen, finden Sie viele unvollständige Zahlungen. Weil der Stripe-Server pi_xxxxxxxxxxxxxxx jedes Mal erstellt, wenn die Bestätigungsseite für den Zen-Warenkorb angezeigt wird. und Stripe-Server erstellt zwei pi_xxxxxxxxxxxxxxx für jede Zahlung. Die Ursache liegt in der Spezifikation von stripe server und Zencart.
  4. Wenn Ihre Website verwendet One-Page Checkout (OPC), du muss addieren Streifen zur kommagetrennten Liste der Zahlungsarten, die eine Bestätigung erfordern (Konfiguration :: One-Page Checkout-Einstellungen :: Zahlungsmethoden, die eine Bestätigung erfordern).

Installation und Upgrade

Upgrade von einer Version vor v3.0.0

v3.0.0 und höher des Zahlungsmoduls ist sehr unterschiedlich als Versionen vor v3.0.0. Es sind keine core-file overwrites oder template-file modifications mehr erforderlich (oder gewünscht).

Beim Upgrade auf v3.0.0 und höher von einer Version vor v3.0.0 müssen Sie zuerst verschiedene Dateien und Bearbeitungen entfernen, die von diesen früheren Versionen erforderlich sind. Siehe in diesem GitHub-Wiki-Artikel für Details.

WICHTIG: Nach dem Ändern des API-Schlüssels oder des Testmodus-Status des payment-module's müssen Sie alle vorhandenen Datensätze in der Datenbanktabelle des payment-module's löschen:

  1. Navigieren Sie zu den admin's Tools :: SQL-Patches installieren.
  2. Geben Sie folgendes in das Textfeld ein und drücken Sie dann auf die Schaltfläche "Absenden":

    ABGESCHNITTENER STREIFEN;

Installieren Sie das Zahlungsmodul

Um eine erste Installation des Zahlungsmoduls durchzuführen:

  1. Entpacken Sie die Modul's Distribution zip-Datei.
  2. Umbenennen der YOUR_TEMPLATE Verzeichnis, das mit dem Namen der aktuell aktiven Vorlage übereinstimmt: /includes/template/YOUR_TEMPLATE.
  3. Kopiert alle Dateien und Unterverzeichnisse aus der Distribution's /includes Verzeichnis zu Ihrer Website's /includes Verzeichnis.
    1. /includes/classes/ajax/zcAjaxStripe.php
    2. /includes/classes/observers/auto.stripe.php
    3. /includes/extra_datafiles/stripe_database_names.php
    4. /includes/languages/english/extra_definitions/lang.stripe_extra_definitions.php
    5. /includes/languages/english/modules/payment/lang.stripe.php
    6. /includes/languages/french/extra_definitions/lang.stripe_extra_definitions.php
    7. /includes/languages/french/modules/payment/lang.stripe.php
    8. /includes/languages/german/extra_definitions/lang.stripe_extra_definitions.php
    9. /includes/languages/german/modules/payment/lang.stripe.php
    10. /includes/languages/japanese/extra_definitions/lang.stripe_extra_definitions.php
    11. /includes/languages/japanese/modules/payment/lang.stripe.php
    12. /includes/modules/pages/account/jscript_stripe_account_form.php
    13. /includes/modules/pages/checkout_confirmation/jscript_stripe_checkout_confirmation.php
    14. /includes/modules/pages/checkout_one/jscript_stripe_checkout_one.php
    15. /includes/modules/pages/checkout_one_confirmation/jscript_stripe_checkout_one_confirmation.php
    16. /includes/modules/pages/checkout_payment/jscript_stripe_checkout_payment.php
    17. /includes/modules/payment/stripepay/vendor/*. *
    18. /includes/modules/payment/stripepay/create.php
    19. /includes/modules/payment/stripe.php
    20. /includes/templates/YOUR_TEMPLATE/css/stylesheet_stripe.css
    21. /includes/checkout.js
  4. Navigieren Sie zu den admin's Modules :: Payment. Sie werden die Stripe-Zahlungen : Kreditkarte Modul hinzugefügt. Wählen Sie dieses Modul und Installieren es.
    • Hinweis: Testmodus ist der Standardwert, wenn dieses Modul installiert ist. Vergessen Sie nach dem Test nicht, den Testmodus auf 'false' einzustellen. Geben Sie den von Stripe bereitgestellten geheimen Schlüssel nicht preis.

Upgrade des Payment Module

Folgen Sie den Schritten A bis C im vorherigen (Installation) Abschnitt.

Test auf der Storefront (Testmodus)

Gehen Sie folgendermaßen vor, um sicherzustellen, dass Sie das Zahlungsmodul korrekt installiert und konfiguriert haben:

  1. Legen Sie einen oder mehrere Artikel in Ihren Warenkorb und starten Sie den Checkout-Prozess.
  2. Wählen Sie das Stripe Payment : Credit Card Zahlungsmodul und fahren Sie mit dem Checkout-Bestätigungsschritt fort.
  3. Das Stripe-Karteneingabeformular wird wie folgt angezeigt. Wenn das Formular nicht angezeigt wird, haben Sie möglicherweise einen falschen API-Schlüssel eingegeben.

  4. Geben Sie die VISA-Kreditkartennummer ein 4242 4242 4242 4242, ein Ablaufdatum, das in der Zukunft liegt, eine zufällige dreistellige Nummer im CVC und die Postleitzahl (falls angefordert). Drücken Sie die Taste "Bestellung bestätigt".
  5. Gehen Sie zu Ihrem Stripe-Dashboard. und klicken Sie auf die Schaltfläche Zahlungen und erfolgreich. Überprüfen Sie die Informationen 's Gesamtbetrag, Kundenname und E-Mail-Adresse.
  6. Bestätige die E-Mail-Adresse und die Bestellsumme auf der Zen-Warenkorb admin's Customers :: Orders page. Vergleichen Sie den Gesamtbetrag, den Kundennamen und die E-Mail-Adresse mit den Stripe-Informationen.

Entfernen Sie das Zahlungsmodul

Navigieren Sie zu den admin's Module :: Bezahlung, wählen Sie die Stripe-Zahlungen : Kreditkarte Modul und klicken Sie auf seine Deinstallieren Taste.

Kunden entfernen' Stripe Records

Die Kreditkarteninformationen des Kunden werden verschlüsselt und in der Datenbank gespeichert, können aber nach Ermessen des Kunden gelöscht werden. Der Kunde kann es mit dem Stripe-Kreditkarteninformationen-Löschknopf auf der Seite "Mein Konto & quot; löschen. Nach dem Löschen wird diese Anzeige nicht mehr angezeigt.
